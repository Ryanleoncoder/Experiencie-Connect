# Indice de Documentacao

`docs/` e a area principal de documentacao do Experience Connect.

Comece por estes arquivos:

- [Referencia da Plataforma](PLATFORM_REFERENCE.md) - estrutura atual do repositorio, superficies de runtime e rotas.
- [Arquitetura](ARCHITECTURE.md) - visao geral dos componentes e fluxos.
- [Origem do Projeto](project-origin.md) - motivacao e objetivos.
- [Politica de Dados](privacy.md) - dados usados pela experiencia de jogo.
- [Seguranca](security.md) - controles e convencoes de seguranca.
- [Security Overview (EN)](SECURITY-OVERVIEW.md) - resumo do modelo de seguranca em ingles.
- [Banco de Dados](database.md) - organizacao das migracoes e regras.

```text
docs/
  README.md              -> ponto de entrada
  PLATFORM_REFERENCE.md  -> mapa da repo e runtimes
  ARCHITECTURE.md        -> componentes e fluxos
  project-origin.md      -> motivacao
  privacy.md             -> modelo de dados
  security.md            -> controles de seguranca
  SECURITY-OVERVIEW.md   -> resumo de seguranca (EN)
  database.md            -> banco e migracoes
```

## Escopo

Esta pasta deve conter documentacao limpa e ativa sobre:

- estrutura do projeto;
- arquitetura da plataforma;
- tratamento de dados;
- seguranca;
- banco de dados;
- referencias estaveis de runtime.

Notas temporarias, rascunhos, logs de debug e materiais de investigacao devem ficar fora desta area publica.
